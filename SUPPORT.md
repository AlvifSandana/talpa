# Support

Thanks for using Talpa.

## Where to Get Help

### 1) Questions and usage help

- Open a GitHub Issue with the `question` label (or nearest available label).
- Include:
  - Talpa version (`talpa --version` if available)
  - OS/distribution and architecture
  - Exact command you ran
  - Relevant output/log snippets (redact private data)

### 2) Bug reports

- Open a GitHub Issue and include reproducible steps.
- Helpful details:
  - Expected behavior
  - Actual behavior
  - Reproduction steps
  - `--debug` output when relevant

### 3) Security issues

Please do **not** open public issues for vulnerabilities.

- Report privately via GitHub Security Advisories.
- See [SECURITY.md](SECURITY.md).

## Before Opening an Issue

- Check existing issues to avoid duplicates.
- Run with `--dry-run` first for destructive flows.
- Validate command flags with `talpa --help` and subcommand help.

## Response Expectations

- Best effort community support.
- Triaging time may vary based on maintainer availability.

## Commercial Support

No commercial support is currently offered.
